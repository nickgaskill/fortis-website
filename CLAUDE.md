# Fortis Contracting Group website

Public marketing site for Fortis Contracting Group, a construction & remodeling company in Central Texas (Austin, Georgetown, Lakeway), owned by Nick Gaskill. Live via GitHub Pages; every push to `main` republishes the site automatically in about a minute.

## Architecture

Plain static HTML/CSS/JS — no framework, no build step. Thirty-six real pages, each a full standalone page: `index`, `services` (an overview hub), the four service-category pages (`interior-remodels`, `outdoor-living`, `home-additions-sunrooms`, `new-construction`), plus `portfolio`, `process`, `reviews`, `contact`, `pool-design-guide` (a virtual flip-book brochure), `design-your-pool` (an interactive estimator), `about`, `blog` (+ one post per article, e.g. `choosing-your-pool-style`), and one page per real project (`cimarron-hills`, `dovetail-lane` — each a hero + write-up + `.ch-grid` gallery with a click-to-open lightbox). `portfolio.html` also has a **"Browse by type" category gallery** (below the featured projects): a grid of `.catg` boxes generated by an inline script from the `#catgData` block — the single edit point. Each `<div data-cat data-label>` in that block holds the category's `<img>`s; the box cover is the first photo, the count is automatic, and clicking opens an inline `.clb` lightbox cycling that category. Empty category divs render a "Photos coming soon" placeholder box. CSS/JS for it are inline in `portfolio.html` (not styles.css). Boxes currently start empty — Nick adds his own photos per category; each `<div data-cat>` carries a commented `<img>` example.

- **Navigation**: a sticky header with the wordmark (a small `logo-mark.png` icon (the F-and-house mark cut from the full logo, transparent background) sits left of the "Fortis Contracting Group" text inside the `.logo` link; `.logo-mark` is sized at `1.5em` so it scales with the wordmark), a "Free consultation" button, and a single **"Menu" button** that click-toggles a **dropdown panel** (`.menu-panel`, `id="menuPanel"`) listing Home, Services (an unclickable `.menu-head` label — the category links below it do the navigating; `services.html` still exists and is reachable from the footer and the home page button) (with Interior Remodels / Outdoor Living / Pools / Pool Design Guide / Home Additions & Sunrooms / New Construction sub-links under `.menu-sub`, each linking to its own dedicated page), Portfolio, Process, Reviews, About, Blog, Contact. `services.html` is an overview hub whose four cards link into those pages; the category pages share one layout (`page-hero` → intro + `.cat-list` of services + `.cat-note` → CTA band). All four build categories (**Interior Remodels**, **Outdoor Living**, **Home Additions & Sunrooms**, **New Construction**) instead show a clickable `.tile-grid` (dark-overlaid `.tile-ph` placeholder + white `.tile-corner` bracket + `.tile-label`) each linking to six sub-pages. Interior: `kitchen-remodels`, `bathroom-remodels`, `custom-cabinetry`, `painting`, `flooring`, `whole-home-renovations`. Exterior: `landscape-design`, `landscaping`, `hardscaping`, `pools`, `outdoor-living-spaces`, `sport-courts`. Home Additions (five tiles, so the grid uses `.tile-grid-5` — 3 on top, 2 wider below, no empty cell): `sunrooms`, `home-additions`, `casitas`, `patio-extensions`, `adus`. New Construction (two tiles, `.tile-grid-2`): `residential-construction`, `commercial-construction`. Every category page uses tiles; the sub-pages use the plain `.cat-list` layout. Sub-pages are reachable only from their parent's tiles (not the menu) and each has a “← Back to …” link. Swap `.tile-ph` for an `<img>` as real photos arrive. `site.js` toggles the panel's `hidden` attribute and closes it on outside-click, Escape, or link-click. The phone number is not in the header — visitors reach it via the "Free consultation" button and the contact page. Under 560px the "Free consultation" button hides so the wordmark and Menu fit.
- The **header and footer are copy-pasted into all thirty-four pages** — if you edit one (add a page link, change the phone number), make the same edit in every HTML file. The `aria-current="page"` marker is set on the current page's nav links.
- `styles.css` — all styling. The palette and fonts are CSS variables in `:root` at the top (`--navy` deep blue, `--gold`/`--gold-bright`, `--paper` warm cream, `--black`). Change a color there and it changes site-wide. The wordmark uses `--wordmark` (Cinzel), loaded from Google Fonts via a `<link>` in each page's `<head>` — the only external dependency. `section[id],[id].svc{scroll-margin-top}` keeps the sticky header from covering a heading when the Services dropdown jumps to an anchor.
- `site.js` — the Menu dropdown toggle (closes on outside-click, Escape, or link click) and scroll-reveal animations.
- **Favicon**: `favicon.svg` is the F-and-house mark on a cream rounded tile (bitmap embedded as a data URI); `favicon-32.png` and `apple-touch-icon.png` (180x180) are downscaled from the same tile. `logo-mark.png` is the transparent cutout used in the header and the design-guide cover; the full lockup (mark + FORTIS + tagline) lives on Nick's Desktop as the source file. All three icon files are linked in every page's `<head>`.
- **Pool Design Guide** (`pool-design-guide.html`): a 10-page virtual booklet (cover, welcome, styles, sizes, spa, add-ons, plaster swatches, tile, Pentair equipment, why-Fortis). Two-page spread with a 3D sheet-turn on desktop, single page with fade on mobile; arrows/keyboard/swipe. Its booklet CSS + flip JS are INLINE in that page (the one exception to styles.css/site.js). Photo slots are `.bslot` placeholders; tile 'Sample' squares await AquaTiles imagery (aquatiles.co — don't hotlink their photos, use trade-program assets). Linked from the menu and a button on pools.html.
- **Design Your Pool estimator** (`design-your-pool.html`): a 6-step wizard (style, size, spa, plaster color + finish tier, waterline tile, add-ons) with a sticky Dream Build summary card and live running estimate; the final step posts the build summary + contact info to the same Formspree endpoint (hidden `dream_build` field). ALL DOLLAR AMOUNTS live in the `PRICING` object at the top of its inline script — THEY ARE PLACEHOLDERS Nick must replace with real pricing. CSS/JS are inline in the page (same exception as the design guide). Nested in the menu under Pools next to the Pool Design Guide. Every option card carries inline SVG sketch art from the `SK` object (drawn in-browser, no image files): pool-shape sketches, size footprints scaled inside a constant dashed yard, a spa diagram, ripple overlays on the plaster swatches, plaster-texture speckles (seeded scatter — avoid plain modular spacing, it produces a visible lattice), waterline-tile pattern mock-ups (our own artwork, NOT AquaTiles photography), and add-on icons. Line art uses `stroke="currentColor"` so it flips navy → gold automatically on selected cards.
- **Blog**: `blog.html` is the index (`.svc` cards, one per post). To add a post: copy `choosing-your-pool-style.html`, replace title/desc/canonical/og/BlogPosting schema and the article body, add a card on `blog.html`, and add a sitemap entry. Posts mark Blog as aria-current.
- The contact form posts to Formspree (`https://formspree.io/f/mpqgewbv`) and emails submissions to Nick.Gaskill@fortistexas.com.

## Conventions

- Brand voice: plain, direct, confident. Tagline: "Built different. Built Fortis." The home hero opens with the Latin meaning of the name — "for·tis — Latin adj. — strong; steadfast; built to last" (the old "From the slab to the last coat of paint" kicker is retired).
- Business facts: phone (512) 743-9690, email Nick.Gaskill@fortistexas.com, Instagram @fortistexas. The four customer reviews are verbatim quotes — don't reword them.
- Gallery projects use real locations/dates. Photo placeholders are `<div class="proj-ph">` blocks; replace with `<img>` tags as real photos are added (compress images to ~200-400 KB before committing).
- **SEO layer** (canonical domain `https://www.fortistexas.com`): every page has a keyword-first `<title>`, meta description, `rel=canonical`, and og: tags; every non-home page carries JSON-LD BreadcrumbList, and the 23 service/category pages add a `Service` schema (provider → the `#business` @id, areaServed → 21 cities). `index.html` keeps the LocalBusiness/GeneralContractor schema. `sitemap.xml` (add new pages there!) and `robots.txt` live at the root. The footer's `.foot-areas` line lists served cities as visible text on every page. Note: canonicals/sitemap point at fortistexas.com, so SEO fully activates once the domain is pointed at GitHub Pages.
- To preview locally, open any `.html` file directly in a browser — no server needed.
