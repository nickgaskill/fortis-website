# Fortis Contracting Group website

Public marketing site for Fortis Contracting Group, a construction & remodeling company in Central Texas (Austin, Georgetown, Lakeway), owned by Nick Gaskill. Live via GitHub Pages; every push to `main` republishes the site automatically in about a minute.

## Architecture

Plain static HTML/CSS/JS — no framework, no build step. Twenty-nine real pages, each a full standalone page: `index`, `services` (an overview hub), the four service-category pages (`interior-remodels`, `exterior-living`, `home-additions-sunrooms`, `new-construction`), plus `portfolio`, `process`, `reviews`, `contact`.

- **Navigation**: a sticky header with the wordmark (a small `logo-mark.svg` icon sits left of the "Fortis Contracting Group" text inside the `.logo` link; `.logo-mark` is sized at `1.5em` so it scales with the wordmark), a "Free consultation" button, and a single **"Menu" button** that click-toggles a **dropdown panel** (`.menu-panel`, `id="menuPanel"`) listing Home, Services (with Interior Remodels / Exterior Living / Home Additions & Sunrooms / New Construction sub-links under `.menu-sub`, each linking to its own dedicated page), Portfolio, Process, Reviews, Contact. `services.html` is an overview hub whose four cards link into those pages; the category pages share one layout (`page-hero` → intro + `.cat-list` of services + `.cat-note` → CTA band). All four build categories (**Interior Remodels**, **Exterior Living**, **Home Additions & Sunrooms**, **New Construction**) instead show a clickable `.tile-grid` (dark-overlaid `.tile-ph` placeholder + white `.tile-corner` bracket + `.tile-label`) each linking to six sub-pages. Interior: `kitchen-remodels`, `bathroom-remodels`, `custom-cabinetry`, `painting`, `flooring`, `whole-home-renovations`. Exterior: `landscape-design`, `landscaping`, `hardscaping`, `pools`, `outdoor-living-spaces`, `sport-courts`. Home Additions (five tiles, so the grid uses `.tile-grid-5` — 3 on top, 2 wider below, no empty cell): `sunrooms`, `home-additions`, `casitas`, `patio-extensions`, `adus`. New Construction (two tiles, `.tile-grid-2`): `residential-construction`, `commercial-construction`. Every category page uses tiles; the sub-pages use the plain `.cat-list` layout. Sub-pages are reachable only from their parent's tiles (not the menu) and each has a “← Back to …” link. Swap `.tile-ph` for an `<img>` as real photos arrive. `site.js` toggles the panel's `hidden` attribute and closes it on outside-click, Escape, or link-click. The phone number is not in the header — visitors reach it via the "Free consultation" button and the contact page. Under 560px the "Free consultation" button hides so the wordmark and Menu fit.
- The **header and footer are copy-pasted into all twenty-nine pages** — if you edit one (add a page link, change the phone number), make the same edit in every HTML file. The `aria-current="page"` marker is set on the current page's nav links.
- `styles.css` — all styling. The palette and fonts are CSS variables in `:root` at the top (`--navy` deep blue, `--gold`/`--gold-bright`, `--paper` warm cream, `--black`). Change a color there and it changes site-wide. The wordmark uses `--wordmark` (Cinzel), loaded from Google Fonts via a `<link>` in each page's `<head>` — the only external dependency. `section[id],[id].svc{scroll-margin-top}` keeps the sticky header from covering a heading when the Services dropdown jumps to an anchor.
- `site.js` — the Menu dropdown toggle (closes on outside-click, Escape, or link click) and scroll-reveal animations.
- **Favicon**: `favicon.svg` is a purpose-built square mark derived from `logo-mark.svg` (navy tile, cream F, gold roofline) — the original logo is 240x208 and navy-on-transparent, which reads poorly at 16px and on dark tabs. `favicon-32.png` and `apple-touch-icon.png` (180x180) are rendered from it as fallbacks; regenerate with `qlmanage -t -s <size> -o . favicon.svg`. All three are linked in every page's `<head>`.
- The contact form posts to Formspree (`https://formspree.io/f/mpqgewbv`) and emails submissions to Nick.Gaskill@fortistexas.com.

## Conventions

- Brand voice: plain, direct, confident. Tagline: "Built different. Built Fortis." The home hero opens with the Latin meaning of the name — "for·tis — Latin adj. — strong; steadfast; built to last" (the old "From the slab to the last coat of paint" kicker is retired).
- Business facts: phone (512) 743-9690, email Nick.Gaskill@fortistexas.com, Instagram @fortistexas. The four customer reviews are verbatim quotes — don't reword them.
- Gallery projects use real locations/dates. Photo placeholders are `<div class="proj-ph">` blocks; replace with `<img>` tags as real photos are added (compress images to ~200-400 KB before committing).
- **SEO layer** (canonical domain `https://www.fortistexas.com`): every page has a keyword-first `<title>`, meta description, `rel=canonical`, and og: tags; every non-home page carries JSON-LD BreadcrumbList, and the 23 service/category pages add a `Service` schema (provider → the `#business` @id, areaServed → 21 cities). `index.html` keeps the LocalBusiness/GeneralContractor schema. `sitemap.xml` (add new pages there!) and `robots.txt` live at the root. The footer's `.foot-areas` line lists served cities as visible text on every page. Note: canonicals/sitemap point at fortistexas.com, so SEO fully activates once the domain is pointed at GitHub Pages.
- To preview locally, open any `.html` file directly in a browser — no server needed.
